## 4. 已修補議題（Closed）
### 4.1 SEC-H1 學生端可取得答案（已修補）
- 描述：學生端取題若回傳完整題目欄位，可能取得 `answer`。
- 修補：建立 `STUDENT_SAFE_QUESTION_SELECT` 投影，不含 `answer`。
- 驗證：靜態檢視投影欄位，確認僅含安全欄位。

### 4.2 SEC-H2 ownership 可繞過作答限制（已修補）
- 描述：以姓名/學號 fallback 可能導致作答 ownership 判定不穩定。
- 修補：學生敏感流程強制 `requireStudentAccount`，並以 `student_account_id` 作唯一 ownership。
- 驗證：`submissions`、`student_sessions` schema 已設 ownership 強約束。

### 4.3 SEC-H3 登入暴力破解風險（已修補）
- 描述：若僅單一全域 limiter，難以阻擋集中攻擊帳號。
- 修補：導入雙層 limiter：
  - IP：15 分鐘 30 次。
  - IP+帳號：15 分鐘 6 次。
- 驗證：程式設定已落地於 `/api/login`、`/api/admin/login`。

### 4.4 SEC-M1 歷史資料完整性不足（已修補）
- 描述：舊資料可能存在 owner 缺漏造成約束無法生效。
- 修補：migration 先回填、清理，再重建強約束表結構。
- 驗證：`student_account_id` 在 `student_sessions` 與 `submissions` 均為 `NOT NULL` 且有 FK。

## 5. 未關閉議題（Open）
### 5.1 SEC-M2 CSP 尚未啟用精細策略
- 狀態：Open（Medium）。
- 證據：`helmet({ contentSecurityPolicy: false })`。
- 風險：XSS 防護深度不足。
- 建議：依頁面資源清單制定 `script-src`、`style-src`、`img-src` 白名單。
- 目標完成日：2026-04-22。

### 5.2 SEC-M3 token 查詢 API 仍可匿名訪問
- 狀態：Open（Medium）。
- 範圍：`/api/submissions/lookup`、`/api/submissions/:id`、`/api/submissions/:id/analysis`。
- 現況：依 `lookup_token` 驗證，可用但仍有 token 外洩風險。
- 建議：
  1. 加入專用 rate limiter。
  2. 增加 token TTL 與輪替策略。
  3. 補充查詢行為審計日誌。
- 目標完成日：2026-04-22。

### 5.3 SEC-L1 安全事件監控仍偏人工
- 狀態：Open（Low）。
- 建議：建立 401/403/429 與上傳失敗率告警。
- 目標完成日：2026-06-30。

### 5.4 SEC-L2 依賴套件持續監控機制不足
- 狀態：Open（Low）。
- 現況：`express-rate-limit` 版本為 `^8.2.1`（可用）。
- 建議：建立每月依賴弱點掃描與升級窗口。
- 目標完成日：2026-06-30。
