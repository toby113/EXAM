## 2. 測試策略
### 2.1 測試目標
1. 驗證需求與實作一致（Verification）。
2. 驗證產品滿足使用情境（Validation）。
3. 驗證權限與資料完整性重構無回歸。

### 2.2 測試範圍
- 登入與授權流程（含快速填入按鈕）。
- 題庫/試卷/作答/交卷流程（含隨機排序、計算題容差）。
- 學生端資料隔離與安全投影（答案欄位不外露）。
- 口說上傳授權與檔案管控。
- 資料完整性約束（DB schema 層）。
- 新功能：試卷複製、批次 AI 批改、PDF 匯出、X-Lookup-Token header 查詢。
- 資安強化：速率限制、CSP、token entropy、timing-safe 比對。

### 2.3 測試類型
| 類型 | 說明 |
| --- | --- |
| 靜態驗證 | 語法檢查、Schema 檢查 |
| API 測試 | 正常與異常路徑、授權與越權情境 |
| 整合測試 | 作答 session 到交卷與分析的端到端流程 |
| 回歸測試 | 既有功能在安全重構後的行為一致性 |
| UAT | PO/SME 業務情境確認 |

## 7. 缺陷管理
### 7.1 缺陷分級標準
| 等級 | 定義 |
| --- | --- |
| High | 資安、授權、資料毀損、核心流程不可用 |
| Medium | 功能可用但行為偏差或風險升高 |
| Low | UI/文案/非核心流程問題 |

### 7.2 缺陷處理流程
`發現 -> 登錄 -> 分析 -> 修補 -> 回歸驗證 -> 關閉`

### 7.3 本輪缺陷摘要（v4.0）
| 類別 | 狀態 |
| --- | --- |
| High | 已關閉：6 項（SQL 插值、token entropy、lookup rate limit、token URL 暴露、密碼升級 timing、CSP 停用） |
| Medium | 已關閉：3 項（clone 回傳 200 改 201、PDF 中文亂碼、randomize_options 未實作） |
| Low | 已關閉：2 項（快速填入按鈕焦點、result/analysis 仍用 query token） |
| 未關閉 | 0 |
