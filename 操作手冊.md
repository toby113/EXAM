## 4. 學生端操作 SOP
### 4.1 登入
1. 開啟 `login.html`。
2. 可點擊「👨‍🏫 老師帳號」或「🎒 學生帳號」快速填入預設帳密（示範用途）。
3. 輸入帳號密碼後點擊登入，系統依角色跳轉至對應頁面。
4. 點擊頁面右上角 🌙 圖示可切換深色模式，設定儲存於瀏覽器 localStorage。

### 4.2 選擇考試與開始作答
1. 在考試列表選擇可作答試卷。
2. 如需存取碼，輸入正確碼。
3. 建立作答 session 後進入試題頁。

### 4.3 作答與進度保存
- 可分題作答並持續保存進度。
- 系統僅顯示題目內容，不顯示答案。
- `我不會` 選項將以專用值紀錄，不計正確。
- 點擊題號旁書籤圖示（🔖）可標記/取消書籤；可過濾僅顯示書籤題。
- 題目排列順序與選項可能依試卷設定隨機亂序，每次取卷順序不同。
- 頁面右上角支援深色模式切換（🌙/☀️）。

### 4.4 交卷
1. 點擊提交。
2. 系統完成評分並回傳結果摘要。
3. 可使用結果頁與分析頁查看個人表現。
4. 成績頁提供「下載 PDF 報告」連結，匯出含中文的個人成績報告。

### 4.4a 計算題批改
- 計算題支援數值容差批改，預設容差範圍由出題者設定（`tolerance` 欄位）。
- 例如正確答案為 3.14，tolerance=0.01，則 3.13~3.15 均視為正確。
- 未設定 tolerance 的計算題採文字比對。

### 4.5 口說題上傳
1. 在口說題介面錄音或上傳音檔。
2. 系統會帶入 `submission_id` 與 `lookup_token`。
3. 若作答狀態或權限不符，上傳將被拒絕。

### 4.6 常見學生端異常
| 情境 | 訊息 | 處理方式 |
| --- | --- | --- |
| 未登入或登入逾時 | 401/請先登入 | 重新登入 |
| 無權限操作他人作答 | 403 | 確認使用本人帳號 |
| 查詢碼錯誤 | 403 | 重新從成績頁取得正確查詢碼 |
| 作答已完成仍嘗試更新 | 400 | 建立新作答或聯絡教師 |

## 5. 教師/管理端操作 SOP
### 5.1 管理登入
1. 進入 `login.html` 選教師/管理者。
2. 輸入帳密，登入後進入 `admin.html`。

### 5.2 題庫管理
- 進行題目新增、編輯、刪除、篩選。
- 建議新增後先進審查流程再上線。
- 批次匯入後需抽驗題型、答案與難度。

### 5.3 試卷管理
1. 建立試卷並加入題目。
2. 設定時長、狀態、時間窗、存取碼、可作答次數。
3. **隨機排序**：勾選「題目隨機順序」可讓每位學生看到不同題目順序；勾選「選項隨機順序」可亂排選擇題選項（系統自動還原正確答案對應，評分不受影響）。
4. 開放前先使用測試帳號試跑。
5. **試卷複製**：在試卷列表點擊「複製」按鈕，系統建立一份草稿副本，可修改後再開放。

### 5.4 成績與分析
- 查看單次成績與全卷統計。
- 使用題目品質分析與能力分析輔助教學調整。
- 匯出 CSV 前先確認資料範圍與權限。

### 5.5 人工批改
1. 進入待批改清單。
2. 填寫 rubric 分數與評語。
3. 儲存後重新檢查總分與狀態。
4. **口說題批改**：批改介面內嵌音訊播放器，可直接聆聽學生錄音後給分，無需另開連結。
5. **批次 AI 批改**：在試卷列表點擊「AI批改」按鈕，系統自動對該試卷所有寫作/口說題未批改答案呼叫 AI 批改。需先設定 `.env` 中的 LLM API 金鑰。

### 5.6 AI 輔助出題
1. 設定可用 API 金鑰。
2. 選定科目、題型、難度與數量。
3. 預覽生成結果並人工審核後再入庫。

### 5.7 PDF 報告匯出
1. 學生在成績頁點擊「下載 PDF 報告」。
2. 系統產生含考試標題、得分、每題詳情與解析的 PDF 檔案。
3. 報告使用系統字型（Microsoft JhengHei），支援中文顯示。
4. 若伺服器無 CJK 字型，PDF 仍可下載但中文可能顯示為方塊；請確保 Windows 環境已安裝 `msjh.ttc`（預設已在 Windows 10/11 內建）。

4. 記錄還原結果與影響範圍。

### 6.4 發布流程
1. 取得核准發布版本。
2. 執行備份。
3. 套用程式與 migration。
4. 執行 smoke test。
5. 發布完成並進行 24 小時監控。

## 8. 安全操作要求
1. 管理帳密不得共用，需定期更換。
2. 不可在未授權裝置保存生產 token。
3. 匯出資料須遵循最小必要原則。
4. 發現疑似越權或資料外洩，需立即升級為 P1。

### 8.1 資安強化項目（v4.0 新增）
| 機制 | 說明 |
| --- | --- |
| Token entropy | lookup_token 使用 32 bytes 隨機值（256 bits），防止暴力猜測 |
| Token 傳輸安全 | 查詢 token 改以 `X-Lookup-Token` HTTP header 傳送，不記錄於 URL log |
| Timing-safe 比對 | token/密碼比對使用常數時間演算法，防止 timing side-channel |
| 速率限制 | API 200 次/分鐘；查詢端點 20 次/15 分鐘；帳號操作 5 次/15 分鐘 |
| Cookie 安全 | Session cookie 設定 `HttpOnly`、`SameSite=Strict`；生產環境自動加 `Secure` |
| CSP | Content-Security-Policy 啟用，script/style/font 白名單限制 CDN 來源 |
| SQL 安全 | 所有 DB 查詢使用 prepared statement；動態欄位名稱採白名單 ternary |

## 11. 常見問題（FAQ）
### 11.1 登入被限制（429）怎麼辦？
- 等待限制窗口結束（15 分鐘）後重試。
- 確認是否有異常程式重複送出登入請求。

### 11.2 學生看不到考卷？
- 檢查試卷是否為 `active`。
- 檢查時間窗、存取碼、作答次數是否符合。

### 11.3 上傳音訊失敗？
- 檢查檔案類型與大小（≤50MB，支援 mp3/wav/ogg/m4a/aac/webm）。
- 檢查作答是否仍為 `in_progress`。
- 檢查是否使用本人帳號與有效查詢碼。

### 11.4 還原後分數不一致？
- 確認 DB 與 uploads 是否使用同一時間點備份。
- 檢查是否有 migration 未完整執行。

### 11.5 PDF 下載後中文顯示為方塊？
- 確認伺服器為 Windows 環境，且已安裝 `C:\Windows\Fonts\msjh.ttc`（Microsoft JhengHei）。
- 若為 Linux，需安裝 Noto CJK 字型：`apt install fonts-noto-cjk`。
- 安裝字型後重啟伺服器即可。

### 11.6 試卷「複製」後如何修改再開放？
1. 複製後新試卷狀態為 `draft`（草稿）。
2. 至「編輯試卷」調整標題、題目、時間設定。
3. 將狀態改為 `active` 後儲存即可開放。

### 11.7 批次 AI 批改失敗或沒有反應？
- 確認 `.env` 中 `LLM_PROVIDER` 與對應 API 金鑰已設定。
- 確認 API 金鑰有效且有剩餘額度。
- 若試卷無寫作/口說題，系統回傳「批改 0 筆」屬正常。

### 11.8 計算題仍被判錯，數值看起來很接近？
- 預設 tolerance=0（完全比對）。
- 請至題目編輯介面設定 `tolerance` 欄位（例如 0.01）。
- 系統比較條件：`|學生答案 - 正確答案| ≤ tolerance`。

- 題目排列順序與選項可能依試卷設定隨機亂序，每次取卷順序不同。
- 頁面右上角支援深色模式切換（🌙/☀️）。

### 4.4 交卷
1. 點擊提交。
2. 系統完成評分並回傳結果摘要。
3. 可使用結果頁與分析頁查看個人表現。
4. 成績頁提供「下載 PDF 報告」連結，匯出含中文的個人成績報告。

### 4.4a 計算題批改
- 計算題支援數值容差批改，預設容差範圍由出題者設定（`tolerance` 欄位）。
- 例如正確答案為 3.14，tolerance=0.01，則 3.13~3.15 均視為正確。
- 未設定 tolerance 的計算題採文字比對。

### 4.5 口說題上傳
1. 在口說題介面錄音或上傳音檔。
2. 系統會帶入 `submission_id` 與 `lookup_token`。
3. 若作答狀態或權限不符，上傳將被拒絕。

### 4.6 常見學生端異常
| 情境 | 訊息 | 處理方式 |
| --- | --- | --- |
| 未登入或登入逾時 | 401/請先登入 | 重新登入 |
| 無權限操作他人作答 | 403 | 確認使用本人帳號 |
| 查詢碼錯誤 | 403 | 重新從成績頁取得正確查詢碼 |
| 作答已完成仍嘗試更新 | 400 | 建立新作答或聯絡教師 |

## 5. 教師/管理端操作 SOP
### 5.1 管理登入
1. 進入 `login.html` 選教師/管理者。

### 5.2 題庫管理

1. 建立試卷並加入題目。
2. 設定時長、狀態、時間窗、存取碼、可作答次數。
3. **隨機排序**：勾選「題目隨機順序」可讓每位學生看到不同題目順序；勾選「選項隨機順序」可亂排選擇題選項（系統自動還原正確答案對應，評分不受影響）。
4. 開放前先使用測試帳號試跑。
5. **試卷複製**：在試卷列表點擊「複製」按鈕，系統建立一份草稿副本，可修改後再開放。

- 查看單次成績與全卷統計。

- 匯出 CSV 前先確認資料範圍與權限。

### 5.5 人工批改
1. 進入待批改清單。
2. 填寫 rubric 分數與評語。
3. 儲存後重新檢查總分與狀態。
4. **口說題批改**：批改介面內嵌音訊播放器，可直接聆聽學生錄音後給分，無需另開連結。
5. **批次 AI 批改**：在試卷列表點擊「AI批改」按鈕，系統自動對該試卷所有寫作/口說題未批改答案呼叫 AI 批改。需先設定 `.env` 中的 LLM API 金鑰。

### 5.6 AI 輔助出題
1. 設定可用 API 金鑰。
2. 選定科目、題型、難度與數量。
3. 預覽生成結果並人工審核後再入庫。

### 5.7 PDF 報告匯出
1. 學生在成績頁點擊「下載 PDF 報告」。

3. 報告使用系統字型（Microsoft JhengHei），支援中文顯示。

## 8. 安全操作要求
1. 管理帳密不得共用，需定期更換。
2. 不可在未授權裝置保存生產 token。
3. 匯出資料須遵循最小必要原則。
4. 發現疑似越權或資料外洩，需立即升級為 P1。

### 8.1 資安強化項目（v4.0 新增）
| 機制 | 說明 |
| --- | --- |
| Token entropy | lookup_token 使用 32 bytes 隨機值（256 bits），防止暴力猜測 |
| Token 傳輸安全 | 查詢 token 改以 `X-Lookup-Token` HTTP header 傳送，不記錄於 URL log |
| Timing-safe 比對 | token/密碼比對使用常數時間演算法，防止 timing side-channel |
| 速率限制 | API 200 次/分鐘；查詢端點 20 次/15 分鐘；帳號操作 5 次/15 分鐘 |
| Cookie 安全 | Session cookie 設定 `HttpOnly`、`SameSite=Strict`；生產環境自動加 `Secure` |
| CSP | Content-Security-Policy 啟用，script/style/font 白名單限制 CDN 來源 |
| SQL 安全 | 所有 DB 查詢使用 prepared statement；動態欄位名稱採白名單 ternary |

## 11. 常見問題（FAQ）
### 11.1 登入被限制（429）怎麼辦？
- 等待限制窗口結束（15 分鐘）後重試。
- 確認是否有異常程式重複送出登入請求。

### 11.2 學生看不到考卷？
- 檢查試卷是否為 `active`。
- 檢查時間窗、存取碼、作答次數是否符合。

### 11.3 上傳音訊失敗？
- 檢查檔案類型與大小（≤50MB，支援 mp3/wav/ogg/m4a/aac/webm）。
- 檢查作答是否仍為 `in_progress`。
- 檢查是否使用本人帳號與有效查詢碼。

### 11.4 還原後分數不一致？
- 確認 DB 與 uploads 是否使用同一時間點備份。
- 檢查是否有 migration 未完整執行。

### 11.5 PDF 下載後中文顯示為方塊？
- 確認伺服器為 Windows 環境，且已安裝 `C:\Windows\Fonts\msjh.ttc`（Microsoft JhengHei）。
- 若為 Linux，需安裝 Noto CJK 字型：`apt install fonts-noto-cjk`。
- 安裝字型後重啟伺服器即可。

### 11.6 試卷「複製」後如何修改再開放？
1. 複製後新試卷狀態為 `draft`（草稿）。
2. 至「編輯試卷」調整標題、題目、時間設定。
3. 將狀態改為 `active` 後儲存即可開放。

### 11.7 批次 AI 批改失敗或沒有反應？
- 確認 `.env` 中 `LLM_PROVIDER` 與對應 API 金鑰已設定。
- 確認 API 金鑰有效且有剩餘額度。
- 若試卷無寫作/口說題，系統回傳「批改 0 筆」屬正常。

### 11.8 計算題仍被判錯，數值看起來很接近？
- 預設 tolerance=0（完全比對）。
- 請至題目編輯介面設定 `tolerance` 欄位（例如 0.01）。
- 系統比較條件：`|學生答案 - 正確答案| ≤ tolerance`。
